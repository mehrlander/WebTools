component

demos

window.component.defineComponent(name, tpl, data) registers a custom element backed by an Alpine x-data factory. Three demos below: a self-contained counter, attribute passing, and two components coordinating via kits/messaging.js instead of knowing about each other directly. (kit source)

1. Self-contained counter

defineComponent('demo-counter',
  () => `<button class="btn btn-sm" @click="inc()" x-text="`Count: ${count}`"></button>`,
  () => ({ count: 0, inc() { this.count++ } })
);
<demo-counter></demo-counter>

2. Reading attributes

defineComponent('demo-greeting',
  attrs => `<div>Hello, <b x-text="name"></b>! (passed start=${attrs.start ?? 0})</div>`,
  attrs => ({ name: attrs.name ?? 'world', start: Number(attrs.start ?? 0) })
);
<demo-greeting name="Ada" start="42"></demo-greeting>

3. Cross-component coordination via messaging

The slider publishes on demo.knob; the gauge subscribes. Neither holds a reference to the other. 

defineComponent('demo-knob',
  () => `<input type="range" min="0" max="100" x-model.number="v">`,
  () => ({ v: 50, onMount() { this.$watch('v', n => messaging.publish('demo.knob', 'change', n)); } })
);
defineComponent('demo-gauge',
  () => `<div class="text-3xl font-bold" x-text="n"></div>`,
  () => ({ n: 0, off: null, onMount() { this.off = messaging.subscribe('demo.knob', (_, n) => this.n = n); } })
);
(updates as you drag)
